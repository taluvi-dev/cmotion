# AI-generated motion-graphics references

External examples worth keeping around as visual reference — what
the wider AI-motion-graphics field is currently producing, what we
might want to be able to render (or visibly out-render), and what
the common visual languages look like.

These are *not* cmotion output and shouldn't be used as branding;
they're inspiration / benchmark material.

---

## `whiteboard-explainer-2chars.png`

Two cartoon-bean figures presenting at a board labelled
**"Rigid Methodology"** (line chart, "structured linear stable")
versus **"Starburst Innovation"** (radial burst, "dynamic
disruptive adaptive"). A red bean watches with a thought bubble:
**"Which path…?"** Soft pastel palette — cream background with
black + burnt-orange blob shapes framing the scene.

Style markers:
- Hand-drawn / sketched aesthetic.
- 2D vector-shape characters with squash-and-stretch limbs.
- Mixed-media composition: characters atop bold abstract blobs.
- Text-heavy slides as a narrative device (typical of AI
  explainer-video output today).
- Soft monochromatic lighting; no strong directional cues.

Why it's here: representative of what current AI tools
(Sora / Veo / Runway / similar) produce when asked for
"explainer video / whiteboard animation." cmotion is aiming
for a *different* register — code-described, deterministic,
typography-driven motion graphics — but this is the visual
expectation many users currently bring to "AI motion graphics."
The gap (and how cmotion's output should *not* look the same)
is informative.
